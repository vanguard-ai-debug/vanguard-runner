## 概述

`ScriptProcessor` 是一个强大的脚本执行处理器，支持在工作流中执行 Python 脚本、表达式和函数。它提供了灵活的数据处理和校验能力，特别适用于复杂的业务逻辑处理和断言校验。

### 主要特性

- ✅ 支持三种脚本类型：`python`、`expression`、`function`
- ✅ 完整的变量引用支持（`${variable}`）
- ✅ 函数自定义参数传递
- ✅ 断言支持（`assert` 语句）
- ✅ 安全性检查（白名单机制）
- ✅ 自动输出捕获

---

## 支持的脚本类型

### 1. Python 脚本 (`type: "python"`)

执行完整的 Python 代码块，支持多行代码、变量定义、条件判断、循环等。

**特点：**
- 使用 `exec()` 执行
- 返回所有定义的变量和执行输出
- 适合复杂的业务逻辑处理

### 2. 表达式 (`type: "expression"`)

执行单个 Python 表达式，直接返回表达式的计算结果。

**特点：**
- 使用 `eval()` 执行
- 直接返回表达式结果
- 适合简单的计算和判断

### 3. 函数 (`type: "function"`)

定义并执行一个 Python 函数，支持自定义参数传递。

**特点：**
- 先定义函数，再执行
- 支持参数传递（位置参数、关键字参数）
- 适合可复用的逻辑封装

## 配置说明

### 基础配置结构

```json
{
  "type": "script",
  "data": {
    "config": {
      "script": "脚本内容",
      "type": "python|expression|function",
      "function_name": "函数名（仅 function 类型需要）",
      "function_args": "函数参数（可选）"
    }
  }
}
```

### 配置字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `script` | string | ✅ | 脚本代码内容 |
| `type` | string | ❌ | 脚本类型，默认 `python`。可选值：`python`、`expression`、`function` |
| `function_name` | string | ❌ | 函数名（`type` 为 `function` 时需要），默认 `execute` |
| `function_args` | any | ❌ | 函数参数，支持字典、列表、单个值或 `null` |

## 使用示例

### 示例 1：Python 脚本 - 数据处理

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "python",
      "script": "
# 获取前置节点的结果
data = input_data
items = data.get('items', [])

# 数据处理
processed_items = []
for item in items:
    if item.get('status') == 'active':
        processed_items.append({
            'id': item.get('id'),
            'name': item.get('name'),
            'price': item.get('price', 0) * 1.1  # 加10%的价格
        })

# 输出结果
result = {
    'total': len(processed_items),
    'items': processed_items
}
print(f'处理了 {len(processed_items)} 个激活的项目')
      "
    }
  }
}
```

**返回结果：**
```json
{
  "stdout": "处理了 5 个激活的项目\n",
  "stderr": "",
  "variables": {
    "data": {...},
    "items": [...],
    "processed_items": [...],
    "result": {
      "total": 5,
      "items": [...]
    }
  }
}
```

### 示例 2：表达式 - 简单计算

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "expression",
      "script": "len(input_data.get('items', [])) * 100"
    }
  }
}
```

**返回结果：** `500`（如果 items 有 5 个元素）

### 示例 3：函数 - 带参数

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "function",
      "function_name": "calculate_discount",
      "function_args": {
        "price": 100,
        "discount_rate": 0.2,
        "min_price": 50
      },
      "script": "
def calculate_discount(price, discount_rate, min_price):
    discounted_price = price * (1 - discount_rate)
    final_price = max(discounted_price, min_price)
    return {
        'original_price': price,
        'discount_rate': discount_rate,
        'discounted_price': discounted_price,
        'final_price': final_price,
        'saved': price - final_price
    }
      "
    }
  }
}
```

**返回结果：**
```json
{
  "result": {
    "original_price": 100,
    "discount_rate": 0.2,
    "discounted_price": 80,
    "final_price": 80,
    "saved": 20
  },
  "stdout": "",
  "stderr": "",
  "function_name": "calculate_discount",
  "variables": {}
}
```

## 变量引用

### 变量引用语法

脚本处理器支持在配置中使用 `${variable}` 语法引用变量，配置会通过 `render_recursive` 自动渲染。

### 可用的变量

#### 1. 前置节点结果

前置节点的结果会自动注入到执行上下文中，可以通过节点ID访问：

```json
{
  "function_args": {
    "data": "${node_id_1}"  // 引用节点 node_id_1 的输出
  }
}
```

或者直接使用 `input_data`（第一个前置节点的结果）：

```json
{
  "script": "
data = input_data
status = data.get('status_code')
  "
}
```

#### 2. 上下文变量

引用工作流中定义的变量：

```json
{
  "function_args": {
    "threshold": "${threshold}",
    "mode": "${validation_mode}",
    "config": {
      "timeout": "${context.config.timeout}"
    }
  }
}
```

#### 3. 完整变量引用示例

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "function",
      "function_name": "validate_response",
      "function_args": {
        "response": "${http_request_result}",
        "expected_status": "${expected_status_code}",
        "timeout": "${context.config.timeout}"
      },
      "script": "
def validate_response(response, expected_status, timeout):
    status_code = response.get('status_code')
    assert status_code == expected_status, f'状态码错误: {status_code}'
    
    response_time = response.get('response_time', 0)
    assert response_time < timeout, f'响应时间超时: {response_time}ms'
    
    return {'validated': True}
      "
    }
  }
}
```

## 函数参数支持

### 参数传递方式

| `function_args` 类型 | 传参方式 | 函数调用形式 | 示例 |
|---------------------|---------|-------------|------|
| `null` / 不提供 | 传入 `execution_context` | `func(execution_context)` | 向后兼容模式 |
| `dict` | 关键字参数 | `func(**function_args)` | `{"key": "value"}` |
| `list` / `tuple` | 位置参数 | `func(*function_args)` | `[1, 2, 3]` |
| 其他类型 | 单个位置参数 | `func(function_args)` | `"string"` 或 `123` |

### 方式 1：关键字参数（推荐）

```json
{
  "function_args": {
    "data": "${input_data}",
    "threshold": 100,
    "strict_mode": true
  }
}
```

对应函数定义：
```python
def process_data(data, threshold, strict_mode):
    # ...
```

### 方式 2：位置参数

```json
{
  "function_args": [100, 200, "add"]
}
```

对应函数定义：
```python
def calculate(a, b, operation):
    # ...
```

### 方式 3：单个参数

```json
{
  "function_args": "${input_data}"
}
```

对应函数定义：
```python
def validate(data):
    # ...
```

### 方式 4：向后兼容（无参数）

如果不提供 `function_args`，函数会接收 `execution_context`：

```json
{
  "type": "function",
  "function_name": "execute"
}
```

对应函数定义：
```python
def execute(ctx):
    input_data = ctx.get('input_data')
    context = ctx.get('context')
    # ...
```

## 断言支持

### 使用 assert 语句

脚本处理器支持 Python 标准的 `assert` 语句进行断言校验。断言失败时会抛出 `AssertionError`，工作流引擎会将其识别为断言失败。

### 基本断言示例

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "python",
      "script": "
data = input_data

# 基本断言
assert data.get('status_code') == 200, f'状态码错误: {data.get(\"status_code\")}'
assert 'data' in data, '响应中缺少data字段'
assert len(data.get('data', [])) > 0, 'data字段为空'
      "
    }
  }
}
```

### 复杂断言示例

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "function",
      "function_name": "validate_api_response",
      "function_args": {
        "response": "${http_result}",
        "expected_status": 200,
        "required_fields": ["code", "data", "message"]
      },
      "script": "
def validate_api_response(response, expected_status, required_fields):
    # 校验HTTP状态码
    status_code = response.get('status_code')
    assert status_code == expected_status, \
        f'HTTP状态码错误: 期望{expected_status}, 实际{status_code}'
    
    # 校验响应时间
    response_time = response.get('response_time', 0)
    assert response_time < 1000, f'响应时间过长: {response_time}ms'
    
    # 校验必需字段
    body = response.get('body', {})
    for field in required_fields:
        assert field in body, f'缺少必需字段: {field}'
    
    # 校验业务状态码
    code = body.get('code')
    assert code == 0, f'业务状态码错误: {code}'
    
    # 校验数据完整性
    data = body.get('data', {})
    items = data.get('items', [])
    assert len(items) > 0, '数据项为空'
    
    # 校验每个数据项的必需字段
    for idx, item in enumerate(items):
        assert 'id' in item, f'数据项[{idx}]缺少id字段'
        assert 'name' in item, f'数据项[{idx}]缺少name字段'
        assert isinstance(item.get('price'), (int, float)), \
            f'数据项[{idx}]的price必须是数字'
    
    return {
        'validated': True,
        'item_count': len(items),
        'response_time': response_time
    }
      "
    }
  }
}
```

### 表达式中的断言

表达式类型通常用于计算，但如果需要断言，建议使用 `python` 类型：

```json
{
  "type": "script",
  "data": {
    "config": {
      "type": "python",
      "script": "
result = len(input_data.get('items', []))
assert result > 0, 'items不能为空'
result  # 最后一个表达式作为返回值
      "
    }
  }
}
```

## 安全限制

### 允许的模块（白名单）

脚本处理器只允许导入以下模块：

```python
allowed_modules = {
    'math', 'random', 'datetime', 'json', 're', 'os', 'sys',
    'time', 'collections', 'itertools', 'functools'
}
```

### 允许的内置函数（白名单）

```python
allowed_builtins = {
    'len', 'str', 'int', 'float', 'bool', 'list', 'dict', 'tuple',
    'set', 'range', 'enumerate', 'zip', 'map', 'filter', 'sorted',
    'min', 'max', 'sum', 'abs', 'round', 'print'
}
```

### 禁止的操作

- ❌ 导入非白名单模块
- ❌ 使用 `exec()` 或 `eval()`（表达式类型除外，使用内置的 eval）
- ❌ 定义类（`class`）
- ❌ 以下划线开头的函数名
- ❌ 访问 `__builtins__`
- ✅ **允许 `assert` 语句**（用于断言校验）

### 安全示例

**✅ 允许：**
```python
import json
import math
import datetime
from collections import defaultdict

result = math.sqrt(100)
current_time = datetime.datetime.now()
```

**❌ 禁止：**
```python
import subprocess  # 不在白名单中
import requests    # 不在白名单中
exec("恶意代码")    # 禁止使用 exec
class MyClass:     # 禁止定义类
    pass
```

## 最佳实践

### 1. 选择合适的脚本类型

- **简单计算** → 使用 `expression`
- **复杂逻辑** → 使用 `python`
- **可复用逻辑** → 使用 `function`

### 2. 使用变量引用

尽量使用 `${variable}` 引用变量，而不是硬编码：

```json
{
  "function_args": {
    "threshold": "${threshold}",  // ✅ 推荐
    "mode": "strict"              // ❌ 硬编码
  }
}
```

### 3. 错误处理

使用断言提供清晰的错误信息：

```python
# ✅ 好的做法
assert status_code == 200, f'状态码错误: 期望200, 实际{status_code}'

# ❌ 不好的做法
assert status_code == 200  # 错误信息不明确
```

### 4. 函数参数命名

使用有意义的参数名，提高可读性：

```python
# ✅ 好的做法
def validate_response(response, expected_status, timeout):
    # ...

# ❌ 不好的做法
def validate(a, b, c):
    # ...
```

### 5. 返回值处理

对于 `python` 类型，明确返回需要的结果：

```python
# ✅ 好的做法
result = {
    'status': 'success',
    'count': len(items)
}
result  # 明确返回

# 或者
return result
```

## 常见问题

### Q1: 如何在脚本中访问前置节点的结果？

**A:** 有几种方式：

1. 通过 `input_data`（第一个前置节点的结果）：
   ```python
   data = input_data
   ```

2. 通过节点ID访问（如果有多个前置节点）：
   ```json
   {
     "function_args": {
       "data1": "${node1}",
       "data2": "${node2}"
     }
   }
   ```

3. 通过 `execution_context`（function 类型，无参数时）：
   ```python
   def execute(ctx):
       data1 = ctx.get('node1')
       data2 = ctx.get('node2')
   ```

### Q2: 表达式类型和 Python 类型有什么区别？

**A:** 
- **表达式类型**：使用 `eval()` 执行单个表达式，直接返回结果
- **Python 类型**：使用 `exec()` 执行代码块，返回所有变量和输出

示例：
```python
# expression 类型
"len(input_data.get('items', []))"  # 直接返回数字

# python 类型
"items = input_data.get('items', [])\ncount = len(items)"  # 返回所有变量
```

### Q3: 函数参数如何引用变量？

**A:** 在配置中使用 `${variable}` 语法，配置会自动渲染：

```json
{
  "function_args": {
    "data": "${input_data}",
    "config": "${context.config}"
  }
}
```

### Q4: 断言失败时会发生什么？

**A:** 
- 抛出 `AssertionError`
- 工作流引擎识别为断言失败
- 节点状态标记为 `failed`
- 错误信息包含在步骤结果中

### Q5: 如何调试脚本？

**A:** 
1. 使用 `print()` 输出调试信息（会被捕获到 `stdout`）
2. 检查返回结果中的 `variables` 字段
3. 查看工作流执行日志

### Q6: 可以在脚本中使用哪些 Python 特性？

**A:** 
- ✅ 基本语法：变量、函数、条件、循环
- ✅ 数据类型：dict、list、tuple、set
- ✅ 白名单模块和函数
- ✅ `assert` 语句
- ❌ 类定义
- ❌ 非白名单模块导入
- ❌ `exec`/`eval`（内置的除外）

## 完整示例

### 完整工作流节点示例

```json
{
  "id": "validate_and_process",
  "type": "script",
  "data": {
    "config": {
      "type": "function",
      "function_name": "process_order",
      "function_args": {
        "order_data": "${get_order_result}",
        "discount_rate": "${discount_rate}",
        "min_price": 50
      },
      "script": "
def process_order(order_data, discount_rate, min_price):
    # 1. 数据校验
    assert 'items' in order_data, '订单缺少items字段'
    assert len(order_data['items']) > 0, '订单items为空'
    
    items = order_data['items']
    total_price = 0
    processed_items = []
    
    # 2. 处理每个订单项
    for item in items:
        # 校验必需字段
        assert 'id' in item, '订单项缺少id字段'
        assert 'price' in item, '订单项缺少price字段'
        
        # 计算折扣价格
        original_price = item['price']
        discounted_price = original_price * (1 - discount_rate)
        final_price = max(discounted_price, min_price)
        
        processed_items.append({
            'id': item['id'],
            'name': item.get('name', ''),
            'original_price': original_price,
            'final_price': final_price,
            'discount': original_price - final_price
        })
        
        total_price += final_price
    
    # 3. 最终校验
    assert total_price > 0, '总价必须大于0'
    
    # 4. 返回处理结果
    return {
        'success': True,
        'order_id': order_data.get('order_id'),
        'item_count': len(processed_items),
        'total_price': total_price,
        'items': processed_items,
        'discount_rate': discount_rate
    }
      "
    }
  }
}
```

- ✅ 强大的断言支持
- ✅ 安全的白名单机制
